How to set up and track a project on GitHub?

1. Create the repository on GitHub.
- Log in to your GitHub account.
- Click the "+" button (top right) and then click New repository.
- Give your repository a name.
- Important: Do not select any initialization options (neither README, .gitignore, nor license). The repository must remain completely empty.
- Click Create repository.
- Copy the provided HTTPS or SSH URL (it will look something like https://github.com/Ncl-cmd/Algorithm.git).

2. Configure your local Git:
- Open your terminal in your local project folder and run the following commands:

	a) Initialize local Git (if it hasn't already been done):
		git init

	b) Add your files and create your first commit:
		git add .
		git commit -m "First commit"

	c) Rename the main branch to "main" (GitHub standard - sometimes it's named 'master'):
		git branch -M main

	d) Link your local folder to GitHub (replace the URL with your own):
		git remote add origin https://github.com/Ncl-cmd/Algorithm.git

	e) Submit your code to GitHub:
		git push -u origin main

Once these steps are complete, refresh your browser page on GitHub to see your files online.
The connection to GitHub can be made using:
	- the HTTPS method (with a token)
	- an SSH key
This needs to be known to avoid authentication error.

	

Once the github and the local git updated, when the local repository changes, what should I do to update github?

To update GitHub after making changes to your local files, you need to run three basic commands in your terminal.Here is the exact workflow to follow every time you want to save and push your progress:

1. Stage your changesThis prepares your modified files for the next snapshot.To add all changes (new, modified, and deleted files):

	git add .			# Add all files for git to include them for tracking.

   Or   git add 'filename' 		# To add a specific file only

2. Commit your changes
This saves the snapshot locally with a descriptive message about what you changed.

	git commit -m "Describe your changes here"

3. Push to GitHub
This sends your local commits to your online GitHub repository.

	git push			

(Note: You only need to type git push. Since you already used -u origin main during the first setup, Git remembers where to send the code).

Shortcut (Optional)
If you only modified existing files (and did not create new ones), you can combine steps 1 and 2 into a single command:

	git commit -am "Fix a bug in the code"
	git push
